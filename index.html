Переопределиние CSS на платформе AGORA

На живых примерах посмотрим, что можно сделать с помощью переопределение стилей сайта, никакой магии, только пара костылей на чистом CSS!

Это не шутка, всё что вы увидите в рамках данной лекции на самом деле жуткие костыли и ни в коем случае не рекомендуется к использованию в продакшн.

Основы HTML

HTML (от англ. HyperText Markup Language — язык гипертекстовой разметки) — стандартизированный язык гипертекстовой разметки документов для просмотра веб-страниц в браузере. Веб-браузер получает HTML документ от сервера и интерпретируют его в визуальное представление, на данный момент поддерживается встраивание практически любого типа информации, тест, звук, графика и т.д.

HTML страница строится из отдельных элементов, которые называются HTML-тегами, они записываются с использованием угловых скобок, браузеры не отображают их, но используют для интерпретации содержимого страницы.

В основном HTML-теги несут специальное значение, например тег <img> служит для встраивания в HTML-документ изображений, <p> обозначает параграф текста, <iframe> позволяет встраивать другие HTML-документы в текущий, и это далеко не полный перечень HTML-тегов.

<!doctype html>
<html lang="ru">
<head>
    <meta charset="utf-8">
    <title>Мета заголовок страницы</title>
</head>
<body>
    Содержимое страницы
</body>
</html>

Есть и более общие элементы, которые не несут в себе какой-то специальный контекст, например, тег <div> служит для разделения содержимого страницы на семантические блоки и по умолчанию никак не влияет на отображение, пока не будет стилизован.

HTML в отличие от XML не чувствителен к ошибкам, например можно использовать несуществующие теги или вкладывать теги друг в друга, которые этого не предполагают, документ в любом случае отработает и каким-то образом отрисуется браузером, но это не значит что так можно делать, спецификация может измениться в любой момент, и есть большой шанс получить непредвиденное поведение, поэтому лучше так не делать.

    нельзя

    Структура HTML-документа представляет из себя дерево, его называют DOM (от англ. Document Object Model), HTML-теги вкладываются друг в друга для получения нужного результата. Наравне с основной структурой HTML-документа существует вспомогательная так называемый теневой DOM (от англ. Shadow DOM), в основном он служит для отрисовки внутренностей HTML-тегов, например, <select> или скроллбар окна браузера, как правило на такие элементы нельзя повлиять извне, или доступ к ним будет сильно ограничен.

    По умолчанию блоки страницы выстраиваются сверху вниз, независимо от своего семантического значения, конечно если это не inline элементы, которые выстраиваются в строку, например ссылку можно разместить внутри текста это поведение HTML-тега <a> по умолчанию, но его можно изменить с помощью языка стилей CSS.

    Основы CSS

    CSS (от англ. Cascading Style Sheets - каскадные таблицы стилей) - это специальный язык разметки стилей, которые могут применяться к HTML-документу, был создан для отделения логики отвечающей за внешний вид от разметки страницы. Он позволяет изменить шрифт, цвет, размер или позицию элементов на странице, а также даёт возможность изменять эти свойства плавно, получая эффект анимирования.

    CSS позволяет изменять внешний вид элементов на HTML-странице в определённых пределах, путём связывания HTML-тега с нужным стилем через так называемый селектор. Селектором может выступать как само название тега, например div, так и любой атрибут a[href=”http://example.com/”], идентификатор #example-id или класс .example-class.

    Механизм селекторов очень гибок, можно выбрать множество элементов, или указать какой-то конкретный, выбрать всех прямых потомков, соседей, чётные или нечётный элементы и ещё много всего.

    В названии CSS, не просто так есть слово каскадные, любой CSS-свойство, которые было определно для родительского элемента, будет действовать и на потомка, если конечно его не перекроет более специфичный стиль.

    Например, текст внутри параграфа должен быть черного цвета, пока его не перекроет какое-то более специфичное правило или дефолтный стиль элемента, определённый в браузере по умолчанию.

    Важно понимать, что правила стилизации применяются сверху вниз, то есть правило которые было определено ниже, имеет больший вес, если конечно у них одинаковая специфичность, иначе будут применяться более специфичные правила.

    Есть способ повысить специфичность определённого свойства, независимо от того, насколько специфичен сам селектор, для этого нужно использовать ключевое слово !important, именно так с восклицательным знаком впереди, но будте аккуратны, более специфичный селектор с большим весом используя !important может его переопределить.

    display - это очень важное свойство, оно влияет на отображение элемента, то как он ведёт себя в общем потоке DOM:

    • none - скрыт и не отображается на странице, включая все вложенные элементы.
    • block - элемент занимает так называемый "этаж", то есть пытается занять всю предоставленную ширину родительского контейнера.
    • inline - выстраивается с соседями в строку, так же на такие элементы не действуют некоторые стили присущие блочным элементам, например внешние вертикальные отступы (margin-top, margin-bottom).
    • inline-block - что-то среднее между block и inline, пытается выстроится с соседями в строку, но при этом ведёт себя как блок, то есть может иметь в том числе вертикальные отступы.
    • flex - сам элемент ведёт себя как блочный, на всех прямых потомков распространяется flexbox layout.
    • grid - сам элемент ведёт себя как блочный, на всех прямых потомков распространяется grid layout.

    Кроме типов отображения названных выше, существуют ещё более специфичные, которые применяются в более узкой диапазоне, например table и list-item.

    position - ещё одно важное свойство, элементы страницы располагаются на экране согласно блочной модели, но если вдруг нужно сместить какой-то элемент, при этом не изменяя его отступы или размер, или сделать так, чтобы его положение не влияло на другие элементы DOM дерева, можно использовать смещение. Всего есть несколько типов position:

    • static - это позиция по умолчанию, смещение никак не учитывается.
    • relative - учитывается смещение и влияет на соседние элементы.
    • absolute - учитывается смещение и не влияет на соседние элементы, смещение считается от ближайшего родителя у которого position не static, в противном случае смещение считается от границ окна.
    • fixed - учитывается смещение, не влияет на соседние элементы, смещение всегда считается от границ окна и как бы фиксирует элемент на экране в заданной позиции.
    • sticky - что-то среднее между relative и fixed, позиция элемента учитывает смещение, считается от ближайшего родителя у которого position не static, но в случае если в контейнере есть элементы большего размера, при скролле страницы элемент начнёт вести себя как fixed в пределах родительского блока.

    На позицию элемента также может влиять свойство transform, например, можно сместить элемент от его текущей позиции, или повернуть, а свойство transition позволяет сделать переход плавным и получить эффект анимации.
